3 Step 3 Reoon Email Verification
📌 Menu → Step 3 — Verify Emails (Reoon)
🎛️ Sub-Menu Options
OptionWhen to use
▶️ StartFirst time — resets progress & starts fresh from the first unverified row
⏩ ResumeAfter stop/timeout — continues from saved row
⛔ StopPause verification
📊 ProgressSee detailed stats
⏱️ Timeout Protection System
  • Runs each batch for max 2 minutes.
  • Saves progress to Script Properties and sets trigger.
  • Trigger auto-resumes after 1 minute.
  • Repeats until complete. Execution Timeouts never happen!
🛡️ Credit Protection System
  • Daily Credits Only: The script programmatically queries the Reoon balance API for each key before initiating verification.
  • Automatic Key Rotation: If the active key has 0 daily credits remaining, the script automatically rotates to the next API key.
  • Protection Safeguard: If all keys run out of daily credits, verification immediately pauses and toasts the user, never consuming your paid lifetime/instant credits.
🔒 Verification Priority and Logic
  • 1st check: firstname@ (Col E) is verified first.
  • Skip condition: If firstname@ returns a status of safe or catch_all, the remaining patterns (firstname.lastname@ and firstinitiallast@) are skipped entirely. It is written immediately as the final email.
  • Fallback check: If not safe/catch_all, firstname.lastname@ (Col G) is verified. If it is safe/catch_all, the last pattern is skipped.
  • Final fallback: If none of the verified patterns are safe or catch_all, the final Email is set to firstname@ and its status is written as the final Verification Status.

📋 Import & Append Rules
  • No status filter: Emails of all verification statuses are imported.
  • Apollo Leads Import: All emails from Apollo Leads are also processed and added as independent rows.
  • De-duplication: Standardized email de-duplication is active globally across all sources.
⏱️ Timeout Protection & Output ID
  • Runs each batch for max 2 minutes (limited to 500 rows per batch) to prevent any execution timeouts.
  • Saves index progress and triggers next batch after 1 minute.
  • Spreadsheet ID: Specify ID in config.gs (EXTERNAL_SPREADSHEET_ID) to export and append unique leads to your existing sheet.
  • Crunchbase Summaries: At completion, the columns How Many Leads and Verification Status in the Main_Crunchbase sheet are filled automatically.
